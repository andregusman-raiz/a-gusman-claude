# Example: Raiz Platform Case Study

This directory contains real-world artifacts from the Raiz Platform project, preserved as a case study and reference implementation of the Claude Code framework.

## Contents

- `research/` -- Platform research, observability plans, and log-driven improvement references
- `diagnostico-cruzado/` -- Cross-diagnostic reports (architecture, database, APIs, security, testing, CI/CD, performance, etc.)
- `skills-workspace/` -- Self-improvement pipeline artifacts (eval cases, iteration logs)
- `ai-state/` -- Session state snapshots and error logs from real agent sessions

## Purpose

These files demonstrate how the framework's agents, skills, and playbooks were applied to a production Next.js + Supabase + Vercel project. They serve as:

1. **Reference examples** for how diagnostics, research, and self-improvement pipelines produce artifacts
2. **Training data** for understanding the level of detail expected in each artifact type
3. **Validation** that the framework works in practice on a real codebase

## Note

These files contain project-specific details and are NOT part of the generic framework. They are provided solely as illustrative examples.
