## Executive Summary

Finnet S/A is a Brazilian financial technology company specializing in cloud-based financial management, process automation, data transmission (EDI/RPA), and payment processing. The company operates as an authorized Payment Institution with direct integration to Brazil's Central Bank (BACEN), connecting enterprises to 120+ financial institutions.

**Key Metrics:**
- **Clients**: 40 of Brazil's 100 largest companies
- **Integration Coverage**: 3.2+ million CNPJs, 120+ banks
- **Annual Processing**: 5.6+ million files, 74.8+ million transactions
- **Transaction Volume**: R$2.1 trillion
- **Credit Operations**: R$31 billion

## 3. CNAB Standards & Processing

### CNAB Overview
**Centro Nacional de Automação Bancária** (National Center for Banking Automation)

Created by FEBRABAN to speed up standardized information exchange between clients and banks.

### CNAB Formats

#### CNAB 400
- **Record Size**: 400 character positions per line
- **Capabilities**:
  - Simple and guaranteed collection portfolios
  - Faster, simpler information flow
  - Fewer potential errors
  - Limited to basic operations
- **Exclusions**: No correspondent bank services, no payment scheduling

#### CNAB 240
- **Record Size**: Four segments of 240 positions each
- **Capabilities**:
  - Multiple titles within single file
  - Correspondent bank services
  - Payment scheduling services
  - Guaranteed and simple collections
  - More comprehensive transaction handling
- **Use Case**: Preferred for complex operations requiring extended functionality

### Finnet's CNAB Support

**Processing Pipeline:**
1. Company creates CNAB files within their system
2. Sends to Finnet via API or VAN
3. Finnet transmits to bank via standardized CNAB format
4. Bank processes and returns confirmation
5. Finnet routes return file back to company
6. Company receives reconciliation data

**Remittance & Return Process:**
- Cedente (issuer) creates boleto and generates remittance file
- File sent to Finnet for bank transmission
- Bank processes and returns confirmation (typically 1 business day)
- Return file routed back through Finnet
- Automatic reconciliation triggered

**Supported Services:**
- Cobrança Registrada (Registered Collection)
- Débito Automático (Automatic Debit)
- Boleto Registrado (Registered Boletos)

## 5. Technical Architecture & Connectivity

### Operating Model

**Payment Institution Status:**
- Authorized Payment Transaction Initiator (ITP) with BACEN
- Direct connection to BACEN directory via API
- Independent infrastructure

**VAN Architecture:**
- Acts as Value Added Network (Rede de Valor Agregado)
- Centralizes file traffic from multiple companies
- Standardizes information to bank formats
- Provides single point of connectivity

### Data Flow Models

#### Model 1: CNAB-based (Traditional)
```
Company ERP → Creates CNAB file → Sends to Finnet VAN
→ Finnet converts/routes to Bank → Bank processes
→ Bank creates return file → Finnet routes back → Company receives
```

#### Model 2: API-based (Modern)
```
Company ERP → Calls Finnet API → Finnet routes to Bank API
→ Bank processes → Returns result via API → Company receives
```

#### Model 3: Hybrid (Simultaneous)
```
Companies use both CNAB and APIs in parallel during modernization transition
```

#### Model 4: Full API-to-API
```
Company → Finnet API → Multiple Bank APIs → Real-time processing
```

### Security & Compliance

**Certifications:**
- **PCI DSS**: Payment Card Industry Data Security Standard certified (since 2014)
- **Technical Controls**: Encryption, firewalls, access controls
- **Operational Controls**: Regular audits by financial institutions
- **Compliance**: Adherence to BACEN guidelines
- **Data Protection**: Multiple safeguards and integrity verification

**Standards Compliance:**
- FEBRABAN CNAB standards
- Brazilian Central Bank (BACEN) regulations
- Open Finance standards
- EDI international standards
