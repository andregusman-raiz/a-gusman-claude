Spawn the `ag-Q-44-benchmark-qualidade` agent as a background subagent to run QAT-Benchmark PDCA cycle autonomously.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-44-benchmark-qualidade`
- `mode`: `bypassPermissions` (agent roda autonomo sem perguntas)
- `run_in_background`: `true` (ciclo PDCA longo, nao bloquear sessao)
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Base URL: [URL from arguments or QAT_BENCHMARK_BASE_URL env var]
Scope: [all | BM-01,BM-02 — default: all]
Mode: [full | standard | rapid | smoke — default: standard]
Dimensions: [all | D1,D3,D8 — default: all]

[any extra user instructions from $ARGUMENTS]

Execute o ciclo PDCA completo seguindo suas instrucoes. Comece pela PLAN phase.
```

## Important

- Do NOT try to execute the benchmark cycle inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with the PDCA report (parity index + actions taken)
- If user provides only a URL, use scope=all, mode=standard, dimensions=all as defaults
- PDCA = Plan (load KB + select scenarios) → Do (dual-run + triple-score) → Check (classify + parity) → Act (update KB)
- Modes: full (~$6-12, 3 judges), standard (~$2-4, 1 judge), rapid (~$0.60-1.20, fixed only), smoke (~$0.12-0.25, 5 core)
