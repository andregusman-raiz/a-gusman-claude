Spawn the `ag-Q-13-testar-codigo` agent as a background subagent to create and run tests.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-13-testar-codigo`
- `mode`: `bypassPermissions`
- `run_in_background`: `true`
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Scope: [files/modules to test, or "full suite"]
Framework: [vitest | jest | playwright — auto-detect if not specified]

[user instructions from $ARGUMENTS]

Crie e execute testes seguindo a policy anti-teatralidade. Cada expect() deve poder falhar em cenario real.
```

## Important

- Do NOT try to write/run tests inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with test results summary
