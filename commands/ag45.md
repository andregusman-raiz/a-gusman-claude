Spawn the `ag-Q-45-criar-cenario-benchmark` agent as a background subagent to create new QAT-Benchmark scenarios.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-45-criar-cenario-benchmark`
- `mode`: `bypassPermissions` (agent roda autonomo sem perguntas)
- `run_in_background`: `true` (design de cenarios, nao bloquear sessao)
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Capability: [what to test — e.g., "tool use", "reasoning", "teaching"]
Count: [number of scenarios — default: 5]
Category: [fixed | rotatable | both — default: rotatable]
Dimensions: [focus dimensions — default: all]
Language: [prompt language — default: pt-BR]
Domain: [educational domain — e.g., "matematica 8o ano"]

[any extra user instructions from $ARGUMENTS]

Crie os cenarios seguindo suas instrucoes. Comece analisando cobertura existente.
```

## Important

- Do NOT try to create scenarios inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with the list of scenarios created
- If user provides only a capability, use count=5, category=rotatable, dimensions=all as defaults
- Fixed scenarios (BM-XX): baseline tracking, NEVER modify existing ones
- Rotatable scenarios (BM-RXXX): anti-contamination pool, large variety
