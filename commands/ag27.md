Spawn the `ag-27-deploy-pipeline` agent as a background subagent to run the full 8-stage deploy pipeline.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-27-deploy-pipeline`
- `mode`: `bypassPermissions`
- `run_in_background`: `true`
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Target: [preview | production — default: preview]
Provider: [vercel | other — auto-detect if not specified]

[user instructions from $ARGUMENTS]

Execute o pipeline completo: preflight -> lint -> typecheck -> test -> build -> deploy -> smoke -> report. Pare se qualquer stage falhar.
```

## Important

- Do NOT try to deploy inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with deploy URL and smoke test results
