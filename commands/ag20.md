Spawn the `ag-20-monitorar-producao` agent as a background subagent to monitor post-deploy production health.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-20-monitorar-producao`
- `mode`: `auto`
- `run_in_background`: `true`
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
URL: [production URL to monitor]
Duration: [minutes to monitor — default: 5]

[user instructions from $ARGUMENTS]

Monitore saude pos-deploy: status codes, response times, error rates. Alerte se degradacao detectada.
```

## Important

- Do NOT try to monitor inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with health report
