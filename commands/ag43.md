Spawn the `ag-Q-43-criar-cenario-ux-qat` agent as a background subagent to create new UX-QAT visual scenarios.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-43-criar-cenario-ux-qat`
- `mode`: `bypassPermissions` (agent roda autonomo sem perguntas)
- `run_in_background`: `true` (criacao de cenario e ciclo longo, nao bloquear sessao)
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Screen: [nome/rota da tela — from arguments]
Type: [dashboard|form-flow|landing-page|navigation|data-table|auth-flow|empty-error-states|custom — from arguments, or infer]
Interacoes: [interacoes criticas — from arguments, or infer from type]
Breakpoints: [override — from arguments, or use defaults]

[any extra user instructions from $ARGUMENTS]

Crie o cenario UX-QAT completo seguindo suas instrucoes. Comece pela Phase 0 (preflight).
```

## Important

- Do NOT try to create the scenario inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with the scenario creation report
- Minimum required: screen. Type can be inferred from route name.
- Output: context.md, interactions.ts, journey.spec.ts, golden screenshots, anti-patterns, config entry
- Special commands: `/ag-Q-43 setup` (init UX-QAT in project), `/ag-Q-43 scan` (detect screens)
