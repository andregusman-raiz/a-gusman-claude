Spawn the `ag-Q-16-revisar-ux` agent as a background subagent to review UX and accessibility.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-16-revisar-ux`
- `mode`: `auto`
- `run_in_background`: `true`
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Pages: [specific pages/components or "all"]

[user instructions from $ARGUMENTS]

Revise UX e acessibilidade. Verifique WCAG, contraste, navegacao por teclado, e responsividade.
```

## Important

- Do NOT try to review UX inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with UX findings and recommendations
