Spawn the `ag-Q-40-testar-qualidade` agent as a background subagent to run QAT PDCA cycle autonomously.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-40-testar-qualidade`
- `mode`: `bypassPermissions` (agent roda autonomo sem perguntas)
- `run_in_background`: `true` (ciclo PDCA longo, nao bloquear sessao)
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Base URL: [URL from arguments or QAT_BASE_URL env var]
Scope: [all | QAT-XX — default: all]
Threshold: [1-10 — default: 6]

[any extra user instructions from $ARGUMENTS]

Execute o ciclo PDCA completo seguindo suas instrucoes. Comece pela PLAN phase.
```

## Important

- Do NOT try to execute the QAT cycle inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with the PDCA report (summary + actions taken)
- If user provides only a URL, use scope=all and threshold=6 as defaults
- PDCA = Plan (load KB) → Do (execute 4 layers) → Check (classify) → Act (update KB)
