Spawn the `ag-38-smoke-vercel` agent as a background subagent to run smoke tests against a Vercel deployment.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-38-smoke-vercel`
- `mode`: `auto`
- `run_in_background`: `true`
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
URL: [Vercel deployment URL]
Checks: [health, auth, api, pages]

[user instructions from $ARGUMENTS]

Execute smoke tests contra o deploy Vercel. Verificar health endpoint, paginas criticas, e API responses.
```

## Important

- Do NOT try to run smoke tests inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete
