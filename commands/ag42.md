Spawn the `ag-Q-42-testar-ux-qualidade` agent as a background subagent to run UX-QAT PDCA cycle autonomously.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-Q-42-testar-ux-qualidade`
- `mode`: `bypassPermissions` (agent roda autonomo sem perguntas)
- `run_in_background`: `true` (ciclo PDCA longo, nao bloquear sessao)
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Base URL: [URL from arguments or UX_QAT_BASE_URL env var]
Scope: [all | screen-name — default: all]
Layers: [all | L1,L2,L4 | L3 — default: all]
Threshold: [1-10 — default: 6]

[any extra user instructions from $ARGUMENTS]

Execute o ciclo PDCA completo seguindo suas instrucoes. Comece pela PLAN phase.
```

## Important

- Do NOT try to execute the UX-QAT cycle inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with the PDCA report (summary + actions taken)
- If user provides only a URL, use scope=all, layers=all and threshold=6 as defaults
- PDCA = Plan (load KB + tokens) → Do (capture + evaluate 4 layers) → Check (classify) → Act (update KB)
- L3 (AI Judge) is optional and costs ~$0.05-0.10/screenshot — use --layers=L1,L2,L4 for free runs
