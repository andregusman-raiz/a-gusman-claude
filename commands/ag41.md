Spawn the `ag-41-criar-cenario-qat` agent as a background subagent to create new QAT scenarios.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-41-criar-cenario-qat`
- `mode`: `bypassPermissions` (agent roda autonomo sem perguntas)
- `run_in_background`: `true` (criacao de cenario e ciclo longo, nao bloquear sessao)
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Feature: [descricao da feature a testar — from arguments]
Persona: [quem usa esta feature — from arguments]
Resultado esperado: [o que o usuario espera — from arguments, or infer]
Cenario ID: [QAT-XX — from arguments, or auto-increment]

[any extra user instructions from $ARGUMENTS]

Crie o cenario QAT completo seguindo suas instrucoes. Comece pela Phase 0 (preflight).
```

## Important

- Do NOT try to create the scenario inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with the scenario creation report
- Minimum required: feature + persona. Other fields can be inferred.
- Output: spec file (4 layers), golden sample, anti-patterns, config entry
