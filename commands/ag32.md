Spawn the `ag-32-due-diligence` agent as a background subagent to evaluate external software for incorporation.

## What to do

Use the **Agent tool** with these parameters:
- `subagent_type`: `ag-32-due-diligence`
- `mode`: `auto`
- `run_in_background`: `true`
- `prompt`: Pass the user's arguments below plus the current working directory

## User arguments

$ARGUMENTS

## Prompt template

Compose the Agent prompt like this:

```
Projeto: [CWD or user-provided path]
Software: [name, repo URL, or package to evaluate]
Criteria: [license | security | quality | maintainability | all — default: all]

[user instructions from $ARGUMENTS]

Execute due diligence completa: licenca, seguranca, qualidade, manutencao, fit com stack atual.
```

## Important

- Do NOT try to evaluate inline — ALWAYS spawn as Agent subagent
- After spawning, confirm to the user that the agent is running in background
- The agent will notify when complete with due diligence report and recommendation
